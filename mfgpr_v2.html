The reason choosing mGPR — The Cost Argument¶

A common misreading of the paper is that mGPR should be cheaper to train than standard GPR. It is not, and the paper itself is explicit about this. mGPR is cheaper in the dataset it requires, not in runtime. In fact Table 5 of Lehdili, Oswald & Nguyen (2025) shows the opposite for training time — fitting the joint multi-fidelity model is slower than fitting a single HF-only GP because the joint covariance is larger and more hyperparameters have to be optimised (e.g. 10 s vs 1 s at $N = 10$).

The real cost in this problem is not the hyperparameter optimisation: it is the number of calls to the expensive high-fidelity pricing engine. Every HF evaluation costs 100 000 Monte Carlo paths, while an LF evaluation costs only 100 paths. The whole point of mGPR is therefore:

At the same budget of expensive HF evaluations, can we get better accuracy by adding cheap LF data?

This is the sense in which mGPR is "cheaper": for a target accuracy it needs fewer HF points than a pure GPR, because the correlated LF data carries most of the shape information. Training time is a secondary cost that we pay once; HF data is the bottleneck that scales with the experiment.

The bottleneck is the HF sampling cost. Each HF evaluation costs 100 000 MC paths; the 200 LF evaluations (100 paths each) together cost the equivalent of 0.2 of one HF evaluation — essentially free. So the fair comparison is at equal HF budget, and that is exactly what Section 3 does.

def price_geom_call_bs(S, K, T, r, sigma, corr_matrix):
    """Closed-form BS price of geometric average call."""
    is_single = (S.ndim == 1)
    if is_single: S = S.reshape(1, -1)
    d_a = S.shape[1]
    sg2 = np.einsum('i,j,ij->', sigma, sigma, corr_matrix) / d_a**2
    sg = np.sqrt(sg2)
    logF = np.sum(np.log(S) + (r - 0.5*sigma**2)*T, axis=1)/d_a + 0.5*sg2*T
    F = np.exp(logF)
    d1 = (np.log(F/K) + 0.5*sg2*T) / (sg*np.sqrt(T))
    d2 = d1 - sg*np.sqrt(T)
    p = np.exp(-r*T) * (F*norm.cdf(d1) - K*norm.cdf(d2))
    return float(p[0]) if is_single else p

def price_geom_call_mc(S_batch, K, T, r, sigma, corr_matrix, n_paths, seed=0):
    """MC price of geometric call."""
    rng = np.random.default_rng(seed)
    n_sc, d_a = S_batch.shape
    L = np.linalg.cholesky(corr_matrix)
    prices = np.empty(n_sc)
    for i in range(n_sc):
        Z = rng.standard_normal((n_paths, d_a)) @ L.T
        S_T = S_batch[i]*np.exp((r-0.5*sigma**2)*T + sigma*np.sqrt(T)*Z)
        G_T = np.prod(S_T, axis=1)**(1.0/d_a)
        prices[i] = np.exp(-r*T)*np.mean(np.maximum(G_T-K, 0.0))
    return prices

def price_geom_call_mc_from_Z(S_batch, K, T, r, sigma, corr_matrix, Z):
    """MC price using pre-generated Z."""
    S_batch, Z = np.asarray(S_batch, np.float64), np.asarray(Z, np.float64)
    n_sc, d_a = S_batch.shape
    prices = np.empty(n_sc, np.float64)
    drift, vol = (r-0.5*sigma**2)*T, sigma*np.sqrt(T)
    for i in range(n_sc):
        S_T = S_batch[i]*np.exp(drift + vol*Z)
        G_T = np.prod(S_T, axis=1)**(1.0/d_a)
        prices[i] = np.exp(-r*T)*np.mean(np.maximum(G_T-K, 0.0))
    return prices

# === CRN data generation ===

def paired_mc_price_one_point(S, K, T, r, sigma, corr_matrix, mc_lf, mc_hf, seed):
    """LF + HF price for one point using common random numbers."""
    d_a = len(S)
    rng = np.random.default_rng(seed)
    L = np.linalg.cholesky(corr_matrix)
    Z = rng.standard_normal((mc_hf, d_a)) @ L.T
    y_lf = price_geom_call_mc_from_Z(S.reshape(1,-1), K, T, r, sigma, corr_matrix, Z[:mc_lf])[0]
    y_hf = price_geom_call_mc_from_Z(S.reshape(1,-1), K, T, r, sigma, corr_matrix, Z[:mc_hf])[0]
    return y_lf, y_hf

def generate_training_data(S_pool, N_hf, N_lf, K, T, r, sigma, corr_matrix,
                           mc_hf=100_000, mc_lf=100, seed=42, verbose=True):
    """Generate multi-fidelity training data with CRN."""
    assert N_hf <= N_lf
    rng = np.random.default_rng(seed)
    idx = rng.choice(S_pool.shape[0], size=N_lf, replace=False)
    X_lf = S_pool[idx].copy()
    X_hf = X_lf[:N_hf].copy()
    y_lf, y_hf = np.empty(N_lf, np.float64), np.empty(N_hf, np.float64)
    L = np.linalg.cholesky(corr_matrix)
    if verbose: print(f"Generating: N_hf={N_hf}, N_lf={N_lf}, mc_lf={mc_lf}")
    t0 = time.time()
    for j in range(N_hf):
        y_lf[j], y_hf[j] = paired_mc_price_one_point(X_hf[j], K, T, r, sigma, corr_matrix, mc_lf, mc_hf, seed+j)
    for j in range(N_hf, N_lf):
        Z = np.random.default_rng(seed+j).standard_normal((mc_lf, X_lf.shape[1])) @ L.T
        y_lf[j] = price_geom_call_mc_from_Z(X_lf[j].reshape(1,-1), K, T, r, sigma, corr_matrix, Z)[0]
    if verbose: print(f"Done in {time.time()-t0:.1f}s | corr = {np.corrcoef(y_lf[:N_hf], y_hf)[0,1]:.4f}")
    return X_hf, y_hf, X_lf, y_lf

# === Standard GPR (sklearn) ===

def fit_gpr_sklearn(X_train, y_train, n_restarts=5):
    """Fit GPR with Matérn 5/2 kernel."""
    scaler = StandardScaler().fit(X_train)
    kernel = (C(1.0, (1e-3, 1e3)) * Matern(length_scale=1.0, length_scale_bounds=(1e-2, 1e2), nu=2.5)
              + WhiteKernel(noise_level=1e-6, noise_level_bounds=(1e-10, 1e-1)))
    gp = GaussianProcessRegressor(kernel=kernel, n_restarts_optimizer=n_restarts,
                                   normalize_y=True, random_state=SEED)
    gp.fit(scaler.transform(X_train), y_train)
    return gp, scaler

def predict_gpr_sklearn(gp, scaler, X_new):
    return gp.predict(scaler.transform(X_new))

def evaluate_surrogate(preds, prices_true, V0):
    """Compute MAPE, VaR 99%, ES 97.5%."""
    mape = np.mean(np.abs(preds - prices_true) / V0)
    losses = V0 - preds
    var_99 = np.percentile(losses, 99)
    es_975 = losses[losses >= np.percentile(losses, 97.5)].mean()
    return mape, var_99, es_975

2. Multi-Fidelity GPR — Methodology (V2)¶

2.1 Model¶

We use the AR(1) auto-regressive multi-fidelity model of Kennedy & O'Hagan (2000), which is also the model used in the reference paper:

$$ f_h(x) \;=\; \rho \, f_l(x) \;+\; \delta(x), $$

where $f_l \sim \mathcal{GP}(0, k_l)$ is the low-fidelity process, $\delta \sim \mathcal{GP}(0, k_\delta)$ is an independent HF discrepancy, and $\rho \in \mathbb{R}$ is the scalar coupling between the two fidelities. Both $k_l$ and $k_\delta$ are Matérn 5/2 kernels, each with its own signal variance and length-scale. A small white-noise term is added on the LF block to absorb the MC noise of the 100-path estimator.

Stacking $\mathbf{y} = [\mathbf{y}_l;\ \mathbf{y}_h]$, the joint covariance is

$$ K(\theta) \;=\; \begin{pmatrix} K_l(X_l,X_l) + \sigma_n^2 I & \rho\, K_l(X_l,X_h) \\[2pt] \rho\, K_l(X_h,X_l) & \rho^2\, K_l(X_h,X_h) + K_\delta(X_h,X_h) \end{pmatrix}, $$

with $\theta = (\sigma_l, \ell_l, \sigma_\delta, \ell_\delta, \rho, \sigma_n)$.

2.2 What changed from V1¶

V1 (previous version) — two-step / profile approach. V1 fitted the LF GP first, then froze its hyperparameters and searched for $\rho$ and the HF discrepancy kernel in a second stage (grid search followed by a profile negative log-likelihood). This decomposition is convenient but has two drawbacks:

  1. the LF hyperparameters are estimated without knowing how they will be used by the HF level, so they can fit LF noise that the HF layer would otherwise down-weight
  2. the coupling $\rho$ ends up unidentifiable when the LF data are noisy — we observed $\rho \to 0$ on the raw LF dataset in V1, and had to pre-smooth (de-noise) the LF observations to recover a sensible value.

V2 (this notebook) — joint negative log-likelihood. V2 throws away the two-step decomposition and estimates all six hyperparameters simultaneously by minimising the joint NLL of $\mathbf{y} = [\mathbf{y}_l;\ \mathbf{y}_h]$ under the full $(N_l + N_h)\times(N_l + N_h)$ covariance $K(\theta)$:

$$ \theta^\star \;=\; \arg\min_{\theta}\; \tfrac{1}{2}\, \mathbf{y}^\top K(\theta)^{-1} \mathbf{y} \;+\; \tfrac{1}{2}\log\lvert K(\theta)\rvert \;+\; \text{const.} $$

We implement this as the JointMFGPR class below. A few practical points:

  1. Pre-computed distances. All four pairwise squared-distance matrices ($X_l X_l$, $X_l X_h$, $X_h X_h$) are computed once at initialisation. Each NLL call only rescales them by the current length-scales and does one Cholesky factorisation of $K(\theta)$. This is what makes the joint optimisation affordable.
  2. Optimiser. We use L-BFGS-B with multiple random restarts on $(\log \sigma_l, \log \ell_l, \log \sigma_\delta, \log \ell_\delta, \rho, \log \sigma_n)$ and keep the restart with the lowest NLL.
  3. LF noise is learned. Because $\sigma_n$ is a free parameter estimated jointly with everything else, V2 handles noisy LF data without any pre-smoothing / de-noising step. This is the main qualitative improvement over V1.
  4. Prediction. At a test point $x_\ast$ the posterior mean of $f_h$ is obtained from the standard conditional Gaussian formula applied to the joint $K(\theta^\star)$; see predict_mgpr below.

2.4 A note on the choice of a custom implementation¶

Well-tested multi-fidelity GPR implementations already exist in the Python ecosystem. In particular, GPy and emukit are widely used, and emukit is the exact library cited by the reference paper (ref. [38]). Using emukit's LinearMultiFidelityModel together with GPy kernels would almost certainly give us lower MAPE and a more numerically stable optimiser out of the box, since these libraries have had years of engineering attention on conditioning, restarts, and gradient computations.

We made a deliberate choice not to rely on those built-ins. The goal of this project is not to reach the absolute best benchmark number. It is to understand why multi-fidelity GPR works and to experience every step of it ourselves. Calling a black-box fit() would have given us a good plot, but it would not have taught us what the joint covariance actually looks like, how the profile likelihood of V1 relates to the full joint likelihood of V2, or why pre-computing the pairwise distance matrices is the single change that makes the joint NLL optimisation affordable. A concrete example is the coupling parameter $\rho$ itself. emukit treats $\rho$ as just another internal hyperparameter of LinearMultiFidelityModel and does not study it as a quantity of interest. For us, on the contrary, $\rho$ is the whole point. It is the number that tells us how strongly HF and LF are correlated, and the fact that V1 gave $\rho > 1$ while V2 gives $\rho \approx 1$ is the main piece of evidence that V2 is a better estimator. We would not even have noticed this issue if we had used emukit, because the library simply never puts $\rho$ in front of the user. All of these points only became clear once we had to derive and code them from scratch.

We are aware that our implementation is not on par with emukit in terms of raw performance. V2's MAPE is slightly higher than what a tuned LinearMultiFidelityModel would give on the same data, and our optimiser is less robust near the boundary of the hyperparameter space. We see this as the honest cost of learning. The upside is that we now know where our implementation is weaker than the library version. For example, we suspect our weakest point is the joint optimiser's sensitivity to the initialisation of $\rho$ and $\sigma_n$, together with the absence of analytic gradients in our NLL. These are concrete starting points for a future improvement of the code, and we would not have been able to identify them if we had used emukit from day one.

In short, we chose understanding over benchmark performance. We would rather ship a slightly less accurate model that we fully understand and can improve, than a state-of-the-art model that we would only know how to call.

# === Joint multi-fidelity GPR — FAST version ===

def _precompute_distances(X1, X2):
    """
    Compute the matrix of squared Euclidean distances between two point sets.

    Uses the identity ||x - y||^2 = ||x||^2 - 2 x^T y + ||y||^2 to avoid
    an explicit O(N^2 d) loop; all operations are vectorised over numpy.

    Parameters
    ----------
    X1 : ndarray of shape (n1, d)
    X2 : ndarray of shape (n2, d)

    Returns
    -------
    sqdist : ndarray of shape (n1, n2)
        Entry (i, j) = ||X1[i] - X2[j]||^2.
    """
    sq1 = np.sum(X1**2, axis=1, keepdims=True)
    sq2 = np.sum(X2**2, axis=1, keepdims=True)
    return sq1 - 2 * X1 @ X2.T + sq2.T


def _matern52_from_sqdist(sqdist, lengthscale):
    """
    Evaluate the Matérn 5/2 kernel from a pre-computed squared-distance matrix.

    k(r) = (1 + sqrt(5) r/l + 5 r^2 / (3 l^2)) exp(-sqrt(5) r/l),
    where r = ||x - x'|| and l is the lengthscale.

    Accepts a pre-computed squared-distance matrix to avoid recomputing
    distances on every hyperparameter evaluation during optimisation.

    Parameters
    ----------
    sqdist      : ndarray of shape (n1, n2)
        Pre-computed ||x_i - x_j||^2 values.
    lengthscale : float
        Positive lengthscale parameter l.

    Returns
    -------
    K : ndarray of shape (n1, n2)
        Kernel matrix with entries k(x_i, x_j).
    """
    D = np.sqrt(np.maximum(sqdist / (lengthscale**2), 0.0))
    s5D = np.sqrt(5.0) * D
    return (1.0 + s5D + 5.0/3.0 * D**2) * np.exp(-s5D)


class JointMFGPR:
    """
    Joint multi-fidelity Gaussian Process Regression via the AR(1) model.

    The model couples a low-fidelity (LF) GP f_l ~ GP(0, k_l) and a high-fidelity
    (HF) correction delta ~ GP(0, k_h) through a scaling factor rho:

        f_h(x) = rho * f_l(x) + delta(x)

    The resulting joint covariance on [X_lf; X_hf] has the block structure:

        K = [ sig2_l * K_l(X_lf, X_lf) + tau2_l * I ,  rho * sig2_l * K_l(X_lf, X_hf)         ]
            [ rho * sig2_l * K_l(X_hf, X_lf)         ,  rho^2 * sig2_l * K_l(X_hf, X_hf)
                                                          + sig2_h * K_h(X_hf, X_hf) + tau2_h * I ]

    where K_l and K_h are Matérn 5/2 kernels with their own lengthscales.

    All pairwise distance matrices are computed once at construction and reused
    across every NLL call, avoiding redundant computation during optimisation.

    Attributes
    ----------
    X_lf_s, X_hf_s : ndarray
        Standardised LF and HF input matrices.
    Nl, Nh, N       : int
        Number of LF points, HF points, and total points.
    y_all           : ndarray of shape (N,)
        Concatenated [y_lf; y_hf] observation vector.
    sqdist_*        : ndarray
        Pre-computed squared-distance matrices between all point pair combinations.
    """
    
    def __init__(self, X_lf_s, X_hf_s, y_lf, y_hf):
        self.X_lf_s = X_lf_s
        self.X_hf_s = X_hf_s
        self.Nl = len(y_lf)
        self.Nh = len(y_hf)
        self.N = self.Nl + self.Nh
        self.y_all = np.concatenate([y_lf, y_hf])
        self.eye_N = np.eye(self.N)
        self.eye_Nl = np.eye(self.Nl)
        self.eye_Nh = np.eye(self.Nh)
        
        # Pre-compute ALL distance matrices (done once)
        self.sqdist_ll = _precompute_distances(X_lf_s, X_lf_s)
        self.sqdist_lh = _precompute_distances(X_lf_s, X_hf_s)
        self.sqdist_hh_l = _precompute_distances(X_hf_s, X_hf_s)  # for LF kernel at HF points
        self.sqdist_hh_h = self.sqdist_hh_l.copy()                 # for HF kernel at HF points
    
    def neg_log_likelihood(self, params):
        """
        Evaluate the negative log marginal likelihood for a given hyperparameter vector.

        The 7-dimensional parameter vector is stored in log-space for positivity
        constraints on variance/lengthscale terms; rho is unconstrained:

            params = [log(ell_l), log(sig2_l), log(tau2_l),
                      log(ell_h), log(sig2_h), log(tau2_h),
                      rho]

        The NLL is computed as:
            NLL = 0.5 * y^T K^{-1} y + sum_i log L_{ii} + const,
        where L is the lower Cholesky factor of K. The constant is omitted
        as it does not affect the location of the optimum.

        Per-call cost:
            - 4 element-wise kernel evaluations (no distance recomputation)
            - 1 Cholesky factorisation on an (Nl+Nh) x (Nl+Nh) matrix
            - 2 triangular solves (via numpy.linalg.solve with the Cholesky factor)

        Returns 1e12 on Cholesky failure (non-PD matrix) or non-finite result,
        acting as a soft barrier to keep the optimiser away from degenerate regions.

        Parameters
        ----------
        params : array-like of shape (7,)
            Hyperparameter vector as described above.

        Returns
        -------
        nll : float
            Negative log marginal likelihood (up to additive constants).
        """
        ell_l  = np.exp(params[0])
        sig2_l = np.exp(params[1])
        tau2_l = np.exp(params[2])
        ell_h  = np.exp(params[3])
        sig2_h = np.exp(params[4])
        tau2_h = np.exp(params[5])
        rho    = params[6]
        
        # Build kernel blocks from pre-computed distances
        K_ll = sig2_l * _matern52_from_sqdist(self.sqdist_ll, ell_l) + tau2_l * self.eye_Nl
        K_lh = rho * sig2_l * _matern52_from_sqdist(self.sqdist_lh, ell_l)
        K_hh = (rho**2 * sig2_l * _matern52_from_sqdist(self.sqdist_hh_l, ell_l)
                + sig2_h * _matern52_from_sqdist(self.sqdist_hh_h, ell_h)
                + tau2_h * self.eye_Nh)
        
        K = np.block([[K_ll, K_lh], [K_lh.T, K_hh]])
        K += 1e-6 * self.eye_N
        
        try:
            L = np.linalg.cholesky(K)
        except np.linalg.LinAlgError:
            return 1e12
        
        alpha = np.linalg.solve(L.T, np.linalg.solve(L, self.y_all))
        nll = 0.5 * self.y_all @ alpha + np.sum(np.log(np.diag(L)))
        return nll if np.isfinite(nll) else 1e12


def fit_mgpr_ar1(X_hf, y_hf, X_lf, y_lf, verbose=False):
    """
    Fit a joint AR(1) multi-fidelity GPR by maximising the log marginal likelihood.

    Optimises 7 hyperparameters via L-BFGS-B with multiple random restarts:

        theta = (ell_l, sig2_l, tau2_l, ell_h, sig2_h, tau2_h, rho)

    where ell_* are Matérn 5/2 lengthscales, sig2_* are signal variances,
    tau2_* are noise variances (nugget terms), and rho is the AR(1) scaling
    coefficient relating HF and LF outputs.

    Input features are standardised (zero mean, unit variance) before fitting;
    the scaler is stored in the returned dictionary for use at prediction time.

    Parameters
    ----------
    X_hf : array-like of shape (Nh, d)
        High-fidelity input locations.
    y_hf : array-like of shape (Nh,)
        High-fidelity observations.
    X_lf : array-like of shape (Nl, d)
        Low-fidelity input locations.
    y_lf : array-like of shape (Nl,)
        Low-fidelity observations.
    verbose : bool, optional
        If True, print optimised hyperparameter values and final NLL.

    Returns
    -------
    model : dict with keys:
        "params"   — optimised hyperparameter vector (length 7, partially log-space)
        "rho"      — scalar AR(1) correlation coefficient
        "scaler"   — fitted StandardScaler for transforming new inputs
        "X_lf_s"   — standardised LF training inputs, shape (Nl, d)
        "X_hf_s"   — standardised HF training inputs, shape (Nh, d)
        "K_inv_y"  — precomputed K^{-1} y vector for O(N) prediction, shape (N,)
        "Nl", "Nh" — number of LF and HF training points
    """
    from scipy.optimize import minimize
    
    X_hf = np.asarray(X_hf, np.float64)
    X_lf = np.asarray(X_lf, np.float64)
    y_hf = np.asarray(y_hf, np.float64).ravel()
    y_lf = np.asarray(y_lf, np.float64).ravel()
    
    scaler = StandardScaler().fit(X_lf)
    X_lf_s = scaler.transform(X_lf)
    X_hf_s = scaler.transform(X_hf)
    
    # Build model with pre-computed distances
    model = JointMFGPR(X_lf_s, X_hf_s, y_lf, y_hf)
    
    if verbose:
        print(f"Joint model: {model.Nl} LF + {model.Nh} HF = {model.N} total")
    
    bounds = [
        (np.log(0.01), np.log(50.0)),   # ell_l
        (np.log(1e-4), np.log(100.0)),   # sig2_l
        (np.log(1e-8), np.log(1.0)),     # tau2_l
        (np.log(0.01), np.log(50.0)),    # ell_h
        (np.log(1e-6), np.log(10.0)),    # sig2_h
        (np.log(1e-8), np.log(1.0)),     # tau2_h
        (-2.0, 2.0),                      # rho
    ]
    
    # --- Data-informed initialisation -------------------------------------------
    # sig2_l is seeded from the empirical LF variance; sig2_h is seeded much smaller
    # since the HF correction is assumed to explain residual structure not in LF.
    y_var = np.var(y_lf)
    x0_base = np.array([
        np.log(1.0),           # ell_l
        np.log(max(y_var, 1e-3)),  # sig2_l
        np.log(1e-4),          # tau2_l
        np.log(1.0),           # ell_h
        np.log(max(y_var * 0.01, 1e-5)),  # sig2_h
        np.log(1e-5),          # tau2_h
        1.0,                   # rho
    ])
    
    rng = np.random.default_rng(SEED)
    best_res, best_nll = None, np.inf
    
    # --- Multi-start L-BFGS-B optimisation --------------------------------------
    # Three restarts with randomised perturbations reduce the risk of converging
    # to a poor local minimum. rho is clipped to [0.3, 1.7] in random starts to
    # avoid degenerate solutions with near-zero or sign-flipped correlations.
    for k in range(3):
        if k == 0:
            x0 = x0_base.copy()
        else:
            x0 = x0_base + rng.normal(0, 0.3, size=7)
            x0[6] = np.clip(x0[6], 0.3, 1.7)
        
        try:
            res = minimize(model.neg_log_likelihood, x0,
                         method='L-BFGS-B', bounds=bounds,
                         options={'maxiter': 200, 'ftol': 1e-6})
            if res.fun < best_nll:
                best_nll, best_res = res.fun, res
        except:
            continue
    
    p = best_res.x
    rho = p[6]
    
    if verbose:
        print(f"  ell_l={np.exp(p[0]):.3f} sig2_l={np.exp(p[1]):.4f} tau2_l={np.exp(p[2]):.2e}")
        print(f"  ell_h={np.exp(p[3]):.3f} sig2_h={np.exp(p[4]):.2e} tau2_h={np.exp(p[5]):.2e}")
        print(f"  rho={rho:.4f} | NLL={best_nll:.2f}")
    
    # --- Pre-compute K^{-1} y at optimum for O(N) prediction -------------------
    # Rebuilding K once at the optimum avoids storing the full (N x N) matrix;
    # only the length-N vector K^{-1} y is retained in memory.
    Nl, Nh = model.Nl, model.Nh
    ell_l, sig2_l, tau2_l = np.exp(p[0]), np.exp(p[1]), np.exp(p[2])
    ell_h, sig2_h, tau2_h = np.exp(p[3]), np.exp(p[4]), np.exp(p[5])
    
    K_ll = sig2_l * _matern52_from_sqdist(model.sqdist_ll, ell_l) + tau2_l * np.eye(Nl)
    K_lh = rho * sig2_l * _matern52_from_sqdist(model.sqdist_lh, ell_l)
    K_hh = (rho**2 * sig2_l * _matern52_from_sqdist(model.sqdist_hh_l, ell_l)
            + sig2_h * _matern52_from_sqdist(model.sqdist_hh_h, ell_h) + tau2_h * np.eye(Nh))
    K = np.block([[K_ll, K_lh], [K_lh.T, K_hh]]) + 1e-6 * np.eye(Nl + Nh)
    K_inv_y = np.linalg.solve(K, model.y_all)
    
    return {
        "params": p, "rho": rho, "scaler": scaler,
        "X_lf_s": X_lf_s, "X_hf_s": X_hf_s, "K_inv_y": K_inv_y,
        "Nl": Nl, "Nh": Nh,
    }


def predict_mgpr(model, X_new, batch_size=5000):
    """
    Predict HF posterior mean at new input locations.

    Computes the GP posterior mean via the pre-stored K^{-1} y vector:

        mu(x*) = k(x*)^T K^{-1} y

    where k(x*) = [k_lf(x*, X_lf) ; k_hf(x*, X_hf)] is the cross-covariance
    vector between the test point and the joint training set, evaluated at the
    HF level of the AR(1) model:

        k_lf(x*, X_lf) = rho * sig2_l * K_l(x*, X_lf)
        k_hf(x*, X_hf) = rho^2 * sig2_l * K_l(x*, X_hf) + sig2_h * K_h(x*, X_hf)

    Predictions are computed in batches of `batch_size` to limit peak memory
    usage while maintaining efficiency for large test sets.

    Parameters
    ----------
    model      : dict
        Fitted model dictionary returned by `fit_mgpr_ar1`.
    X_new      : array-like of shape (n, d)
        Test input locations (in original, un-standardised space).
    batch_size : int, optional
        Number of test points per prediction batch. Default: 5000.

    Returns
    -------
    out : ndarray of shape (n,)
        Predicted HF posterior means at each test location.
    """
    p = model["params"]
    ell_l, sig2_l = np.exp(p[0]), np.exp(p[1])
    ell_h, sig2_h = np.exp(p[3]), np.exp(p[4])
    rho = p[6]
    
    scaler = model["scaler"]
    X_lf_s, X_hf_s = model["X_lf_s"], model["X_hf_s"]
    K_inv_y = model["K_inv_y"]
    
    X_s = scaler.transform(np.asarray(X_new, np.float64))
    n = X_s.shape[0]
    out = np.empty(n, np.float64)
    
    # Pre-compute training-side squared norms (reused across batches)
    sq_lf = np.sum((X_lf_s)**2, axis=1, keepdims=True)
    sq_hf = np.sum((X_hf_s)**2, axis=1, keepdims=True)
    
    for s in range(0, n, batch_size):
        e = min(s + batch_size, n)
        Xb = X_s[s:e]
        sq_b = np.sum(Xb**2, axis=1, keepdims=True)
        
        # Compute distances from batch test points to all LF and HF training points
        sqdist_b_lf = sq_b - 2 * Xb @ X_lf_s.T + sq_lf.T
        sqdist_b_hf = sq_b - 2 * Xb @ X_hf_s.T + sq_hf.T
        
        # Cross-covariance vectors at the HF level of the AR(1) model
        k_lf = rho * sig2_l * _matern52_from_sqdist(sqdist_b_lf, ell_l)
        k_hf = (rho**2 * sig2_l * _matern52_from_sqdist(sqdist_b_hf, ell_l)
                + sig2_h * _matern52_from_sqdist(sqdist_b_hf, ell_h))
        
        # Posterior mean: mu(x*) = k(x*)^T K^{-1} y
        out[s:e] = np.hstack([k_lf, k_hf]) @ K_inv_y
    
    return out

gp, scaler = fit_gpr_sklearn(X_train, y_train)              # GPR

ES_975  = losses[losses >= np.percentile(losses, 97.5)].mean()

The surrogate replaces Step 3 of Algorithm 1 in the paper: instead of $M = 100{,}000$ calls to the expensive pricing engine, we use the trained surrogate for essentially free. With $N = 50$ training points, the speed-up is $M/N = 2000\times$.

Comment on VaR and ES¶

For $N_h \geq 50$, both GPR and V2 mGPR recover the true VaR 99 % and ES 97.5 % to within a few basis points of $V_0$, and the two methods are essentially indistinguishable. This is expected, since the HF dataset alone is already dense enough to resolve the tail.

The interesting regime is $N_h \in \{10, 20\}$. There the HF-only GPR underestimates both risk metrics (its posterior mean over-smooths the tail of the loss distribution), while V2 mGPR stays close to the true values because the 200 LF points anchor the shape of the price surface in regions where no HF point has been sampled. This is exactly the same pattern reported in Table 5 of the paper: mGPR's main value is in the small-HF-budget regime, which is the regime that matters in practice for end-of-day FRTB re-pricing where HF calls are capped.

One caveat: because V2's MAPE is marginally higher than V1's, its VaR/ES at $N_h = 10$ are also very slightly noisier than V1's. The differences are at the fourth decimal of VaR, irrelevant for a market-risk use case but worth noting for completeness.

4. Discussion¶

4.1 mGPR is cheaper in data, not in runtime¶

This needs to be stated clearly because it was the main point of confusion during the meeting. mGPR does not train faster than GPR — the joint covariance is larger and the optimiser has more hyperparameters to handle. What mGPR reduces is the number of expensive HF evaluations needed to reach a given accuracy. The 200 LF points used here cost less than 1 % of the total MC-path budget, so adding them is essentially free; the dataset we care about is the HF one, and mGPR makes that dataset go further.

4.2 V2 vs V1 — what we lose and what we gain¶

The figure below shows the V1 result (previous notebook):

Comparing V1 and V2 on the same 5-asset geometric call:

Aspect V1 (two-step, de-noised LF) V2 (joint NLL, raw LF)

The headline MAPE of V2 is marginally higher than V1 — roughly 0.2 percentage points at small $N_h$. In our view this is an acceptable trade-off for three reasons:

  1. The absolute gap is tiny compared with the improvement over HF-only GPR, which is the comparison that actually matters for the cost argument. At $N_h = 10$–20 both V1 and V2 are far below the MC reference line; the 0.2 % difference between them is well inside the Monte Carlo noise of the problem.
  2. V2 recovers a statistically meaningful $\rho$. In V1 we consistently obtained $\rho > 1$ (around 1.2–1.35 on the plot above), which is a symptom of the two-step estimator overfitting the coupling because the LF GP had already been frozen. V2's $\rho$ hovers just above 1 — consistent with the paper's value and with the physical interpretation that HF and LF prices are nearly the same process up to MC noise.
  3. V2 no longer needs the de-noising hack. In V1 we had to smooth the LF observations before fitting, otherwise $\rho$ collapsed to 0. V2 estimates the LF noise level $\sigma_n$ jointly with everything else, so the pipeline is shorter and has one fewer tuning knob.

In short: V2 gives up a small amount of raw accuracy in exchange for a cleaner, more faithful estimator of the multi-fidelity model.

5. Conclusion¶

Version 2 of our multi-fidelity GPR replaces the two-step / profile-likelihood estimator of V1 with a single joint optimisation of the full AR(1) model over all six hyperparameters, using our own JointMFGPR class with pre-computed distance matrices for speed. The main takeaways from this revision are:

  1. Methodology is cleaner. V2 fits one joint negative log-likelihood instead of freezing the LF GP and then searching for $\rho$. LF noise is learned as a parameter, so the LF de-noising step that V1 required is no longer needed.
  2. $\rho$ is now identifiable and physically sensible. V2 gives $\rho \approx 1$, in line with the paper; V1 returned $\rho > 1$, which was a sign of the two-step estimator overfitting the coupling.
  3. A small accuracy cost, for a better model. V2's MAPE is about 0.2 percentage points higher than V1's at small $N_h$. This is a small price for
    1. getting rid of the de-noising hack and
    2. obtaining a believable $\rho$. The ranking GPR $\gg$ mGPR at small $N_h$ is preserved.
  4. The cost argument of the paper is confirmed. mGPR is not faster to train than GPR — it is slower. But at equal HF budget it is more accurate, and at equal accuracy it needs fewer HF evaluations. That is the sense in which mGPR is cheaper: in the only resource that actually scales, which is the call count to the expensive pricing engine.
  5. Risk metrics. For $N_h \leq 20$, V2 mGPR tracks the true VaR 99 % and ES 97.5 % noticeably better than HF-only GPR; for $N_h \geq 50$ the two methods agree. This is the regime where a multi-fidelity surrogate is worth the added implementation complexity.

Overall V2 is a more principled implementation of the same idea than V1, trading a marginal MAPE loss for a much more trustworthy estimate of the multi-fidelity coupling and a simpler LF pipeline.
